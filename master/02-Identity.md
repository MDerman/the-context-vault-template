# 02-Identity

Identity comes before Momentum. It defines what each entity is, what it values, what it is trying to become, and what agents should preserve when they create tasks or summarize priorities.

Each embedded section below comes from the entity's root `DECLARATION.md`.

## How To Use

- Keep durable identity in each `DECLARATION.md`.
- Keep time-bound goals inside daily, weekly, quarterly, and yearly periodic notes.
- Link to supporting notes instead of copying long thought-process sections into declarations.

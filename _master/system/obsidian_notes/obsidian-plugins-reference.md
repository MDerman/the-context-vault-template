
# Obsidian Plugins Reference

This note explains the important Obsidian plugins and configured workflows in this vault. For hotkeys, see [[obsidian-keyboard-shortcuts]]. For plain Obsidian fundamentals, see [[obsidian-core-features]].

## TaskNotes

TaskNotes is the primary task system. It stores one Markdown file per task.

Configured root-vault behavior:

- Task folder: `{{context}}/_obsidian/tasks`
- Default context: `personal`
- Archive folder: `{{context}}/_obsidian/tasks/archive`
- Task identification: `task` tag in YAML frontmatter
- Default priority: `normal`
- Default status: `backlog`
- Default scheduled date: none
- Date fields: `scheduled` is the start/surface/work-on date; `due` is the true deadline.
- Time fields: `timeEstimate` is estimated effort in minutes; `timeEntries` stores actual tracked time; `pomodoros` stores Pomodoro tracking.

In practice:

- A task with context `impression` is created in `business/_obsidian/tasks`.
- A task with no context defaults to `personal/_obsidian/tasks`.
- Prefer contexts for context folder routing and tags for sub-project/grouping.
- Use `due` and `scheduled` exactly in task frontmatter. Do not rename them to `due_date` or `scheduled_date`; Bases and generated agent context expect the TaskNotes native names.
- Do not add `scheduled` by default. Add it only when a task should appear on a specific work date.
- Use `timeEstimate`, not `duration`, for estimated effort. Use TaskNotes time tracking to create `timeEntries` for actual time.
- `Alt+Cmd+T` creates a new task.
- Right-click a task to start time tracking.

Google Calendar workflow:

- Native TaskNotes Google export is intentionally not the primary sync path because it supports one target task calendar. The vault mirror uses separate calendars.
- `Time Blocks` is for broad manual or AI-created planning blocks.
- `Scheduled Tasks` mirrors TaskNotes `scheduled`.
- `Due Tasks` mirrors TaskNotes `due`.
- First-time setup: run `gws auth setup`, then `gws auth login --scopes calendar,drive`, then `vault gcal calendars ensure --apply`.
- `vault gcal calendars ensure --apply` sets default popup reminders: `Time Blocks` 0 minutes before, `Scheduled Tasks` 0 minutes before, and `Due Tasks` 0 plus 25 minutes before.
- Google Calendar's API does not expose the UI-only all-day default reminder time such as "0 days before at 9:00 AM"; set that manually in Google Calendar if needed.
- Use `vault gcal list --days 7 --calendar all --json` to inspect upcoming calendar context.
- Use `vault gcal create-block ... --apply` to create broad work blocks on `Time Blocks`.
- Use `vault gcal sync-tasks --dry-run` before `vault gcal sync-tasks --apply`.
- Context Nine runs `vault gcal sync-tasks --apply` every 5 minutes while Obsidian is open.

Useful TaskNotes natural-language input:

| Syntax | Meaning |
| --- | --- |
| `#tag` | Add tag. |
| `@context` | Set context, for example `@business`. |
| `+project` or `+[[Project Name]]` | Set project. |
| `open`, `in-progress`, `done` | Set status. |
| `high`, `normal`, `low` | Set priority. |
| `tomorrow`, `next Friday`, `January 15 at 3pm` | Parse dates/times. |
| `2h`, `30min`, `1h30m` | Set time estimate. |
| `daily`, `weekly`, `every Monday` | Set recurrence. |

The `*` status trigger is enabled. The `!` priority trigger exists but is disabled in current settings.

## Periodic Notes And Calendar

Periodic Notes owns dated notes, including blank monthly SOP notes surfaced from the generated dashboard.

Root vault defaults point at `personal`:

| Cadence | Folder | Format |
| --- | --- | --- |
| Daily | `personal/_obsidian/periodic/daily` | `YYYY-MM-DD` |
| Weekly | `personal/_obsidian/periodic/weekly` | `GGGG-[W]WW` |
| Monthly | `personal/_obsidian/periodic/monthly` | `YYYY-MM` |
| Quarterly | `personal/_obsidian/periodic/quarterly` | `YYYY-[Q]Q` |
| Yearly | `personal/_obsidian/periodic/yearly` | `YYYY` |

Context folder notes are the source of truth. Master rollups live under `_master/_obsidian/periodic` and are generated by:

```bash
python3 _master/system/scripts/periodic.py
```

Calendar gives sidebar navigation for daily notes.

Context Nine auto-routes newly pasted/dropped files from `_master/_obsidian/attachments/_inbox` every 60 seconds while Obsidian is open.

## Templater

Templater is enabled on file creation and routes new files by folder.

Important settings:

- Shared templates folder: `_master/_obsidian/templates/shared`
- User scripts folder: `_master/system/scripts`
- Folder templates enabled
- System commands disabled
- `Alt+R`: replace templates in the active file

Context folder periodic folders use their local templates:

```text
<context-folder>/_obsidian/periodic/daily      -> <context-folder>/_obsidian/templates/periodic/daily-template.md
<context-folder>/_obsidian/periodic/weekly     -> <context-folder>/_obsidian/templates/periodic/weekly-template.md
<context-folder>/_obsidian/periodic/monthly    -> <context-folder>/_obsidian/templates/periodic/monthly-template.md
<context-folder>/_obsidian/periodic/quarterly  -> <context-folder>/_obsidian/templates/periodic/quarterly-template.md
<context-folder>/_obsidian/periodic/yearly     -> <context-folder>/_obsidian/templates/periodic/yearly-template.md
<context-folder>/_obsidian/tasks               -> _master/_obsidian/templates/shared/default-tasks-template.md
```

Useful Templater snippets:

```md
<% tp.file.title %>
<% tp.date.now("YYYY-MM-DD") %>
<% tp.file.folder(true).split('/')[0] %>
```

## Sync Embeds

Sync Embeds makes embedded notes editable in place. Master periodic rollups use it so context folder notes can be edited directly from the master rollup.

````md
```sync
![[personal/_obsidian/periodic/daily/2026-05-10]]
```
````

Further reading: [[README-sync-embeds]]

Sync Embeds supports normal editing shortcuts inside embeds and has header helpers such as `Alt+2` through `Alt+6`.

## Omnisearch

Omnisearch is the preferred fast search layer for this vault. It is installed from Community Plugins as `omnisearch`.

Use `Shift then Shift` for `Omnisearch: Vault search` when finding a document across the vault. Doubleshift handles that double-tap shortcut, and `Cmd+Shift+F` remains a fallback. Use `Cmd+F` for `Omnisearch: In-file search` when skimming matches inside the active Markdown note. Core Global Search is still available on `Cmd+Shift+P` for cases where the built-in search syntax is needed.

Omnisearch is different from core Global Search: it is optimized for quickly finding the most relevant notes, not for replacing every advanced core-search workflow. It ranks documents by terms in filenames, paths, headings, and repeated note content, and it tolerates small typos.

Useful query filters:

- `path:"somepath"` restricts results to matching paths.
- `ext:"png jpg"`, `ext:png`, or `.png` filters by file type.
- `"exact expressions"` in quotes narrows results.
- `-exclusions` removes notes containing excluded words.

From Vault search, press `Tab` on a result to inspect matches inside that note. In-file search jumps to the selected match with `Enter`.

PDF, Office document, and image indexing require Text Extractor. Those extra indexing modes are not currently enabled.

## Open Tab Settings

Open Tab Settings controls how notes open in Obsidian tabs. It is installed from Community Plugins as `open-tab-settings`.

Configured behavior:

- Always open files in a new tab by default.
- Prevent duplicate tabs: if a file is already open, switch to its existing tab instead of opening another copy.
- Deduplicate across tab groups, split panes, and popup windows.
- New tab placement: after the active tab.
- Split workspace placement: open new tabs in the same tab group.
- `Cmd+Click` and middle-click behavior: open links in the opposite tab group when possible.
- Focus explicit new tabs is disabled, so explicit `Cmd+Click` and middle-click opens can stay in the background; regular clicks still focus the opened tab.

## Excalidraw

Excalidraw is configured for drawings and visual thinking.

Root vault paths:

- Drawings folder: `_master/_obsidian/excalidraw`
- Template: `_master/_obsidian/excalidraw/template.excalidraw`
- Scripts: `_master/_obsidian/excalidraw/Scripts`
- Font assets: `_master/_obsidian/excalidraw/CJK Fonts`

Context folders use local `_excalidraw` folders, with scripts copied from the master source during bootstrap/sync workflows.

## Bases

Bases provide file/property dashboards. Master dashboards live in `_master/_obsidian/bases`, including:

- `_master/_obsidian/bases/tasks-today.base`
- `_master/_obsidian/bases/tasks-this-week.base`

Each context folder has a local dashboard at `<context-folder>/_obsidian/bases/context-dashboard.base`.

## Workflows From Older Plugin Notes

- Workspaces: save a layout with the Workspaces command, then reload it later from the Workspaces modal.
- Bookmarks: bookmark frequently used notes or headings, then organize them in Bookmark folders.
- Note Composer: select text, right-click, and extract it into a new or existing note.
- Outline: turn on Outline and keep it in the sidebar for long notes.
- Kanban: if enabled later, create boards from the `Kanban: Create new board` command and link cards to notes/dates.
- iFrames: embed web views with HTML only when needed:

```html
<iframe src="URL" style="width: 100%; height: 500px;"></iframe>
```
